服务器初始化清单（阿里云 ECS，Ubuntu 22.04）                                                                                                                                                     
   
  1. 安装基础软件                                                                                                                                                                                  
                                                            
  # Docker
  curl -fsSL https://get.docker.com | sh
  systemctl enable --now docker

  # Docker Compose plugin
  apt-get install -y docker-compose-plugin

  # sqlx-cli（用于执行数据库迁移）
  curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
  source ~/.cargo/env
  cargo install sqlx-cli --no-default-features --features postgres

  2. 创建目录结构

  mkdir -p /opt/playmate/{bin,migrations,logs}
  chmod 750 /opt/playmate

  3. 配置环境变量

  # 复制模板后编辑填入真实值
  cp /path/to/repo/infra/.env.prod.example /opt/playmate/.env.prod
  vim /opt/playmate/.env.prod   # 填入 POSTGRES_PASSWORD、JWT_SECRET 等
  chmod 600 /opt/playmate/.env.prod

  ▎ 注意：DATABASE_URL 中的密码必须与 POSTGRES_PASSWORD 一致。

  4. 启动 PostgreSQL + Redis

  cd /opt/playmate
  # 将 docker-compose.prod.yml 上传到此目录（或 scp 过来）
  docker compose -f docker-compose.prod.yml --env-file .env.prod up -d
  docker compose -f docker-compose.prod.yml ps   # 确认两个服务 healthy

  5. 配置 systemd 服务

  cat > /etc/systemd/system/playmate.service << 'EOF'
  [Unit]
  Description=Playmate Gateway
  After=network.target docker.service
  Requires=docker.service

  [Service]
  Type=simple
  User=root
  WorkingDirectory=/opt/playmate
  EnvironmentFile=/opt/playmate/.env.prod
  ExecStart=/opt/playmate/bin/playmate-gateway
  Restart=always
  RestartSec=5
  StandardOutput=append:/opt/playmate/logs/app.log
  StandardError=append:/opt/playmate/logs/app.log

  [Install]
  WantedBy=multi-user.target
  EOF

  systemctl daemon-reload
  systemctl enable playmate

  6. 配置 GitHub Actions Secrets

  在 GitHub 仓库 → Settings → Secrets and variables → Actions 添加：

  ┌────────────────┬──────────────────────────────────────────┐
  │     Secret     │                    值                    │
  ├────────────────┼──────────────────────────────────────────┤
  │ SERVER_HOST    │ 服务器公网 IP                            │
  ├────────────────┼──────────────────────────────────────────┤
  │ SERVER_SSH_KEY │ 服务器 SSH 私钥内容（cat ~/.ssh/id_rsa） │
  └────────────────┴──────────────────────────────────────────┘

  确保对应公钥已在服务器 ~/.ssh/authorized_keys 中。

  7. 首次手动部署（验证流程）

  # 在本地交叉编译（或 CI 跑完后手动上传）
  scp target/x86_64-unknown-linux-gnu/release/playmate-gateway root@SERVER:/opt/playmate/bin/
  scp infra/migrations/* root@SERVER:/opt/playmate/migrations/

  # 在服务器上
  set -a && source /opt/playmate/.env.prod && set +a
  sqlx migrate run --source /opt/playmate/migrations --database-url "$DATABASE_URL"
  chmod +x /opt/playmate/bin/playmate-gateway
  systemctl start playmate
  systemctl status playmate
  curl http://localhost:8080/health

  8. 后续推送自动触发

  首次初始化完成后，每次 git push origin main 将自动：编译 → 上传 → 迁移 → 重启 → 健康验证。
